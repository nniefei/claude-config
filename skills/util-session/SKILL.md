# 会话上下文保存 Skill

> 将当前会话的工作上下文精简保存到系统提供的项目 memory 路径下的 `sessions/` 子目录。每次保存生成独立文件，支持手动读取指定文件以接续工作。
> 以下简写 `memory/` 均指系统提供的项目 memory 路径（每次会话系统会自动告知当前路径）。
> 定位：**工作快照**——记录"做到哪了"以便下次续接。

## 使用方式

| 命令 | 说明 |
|:---|:---|
| `/util-session save` | 立即保存一次 |
| `/util-session load [文件名]` | 读取指定的 session 文件，恢复上下文 |
| `/util-session list` | 列出所有已保存的 session 文件 |
| `/util-session` | 无参数时等同于 `save` |

## 执行步骤

### 子命令：`save`（也是无参数默认行为）

执行"保存流程"（见下方），完成后告知用户：
```
💾 会话上下文已保存喵！
文件: memory/sessions/YYYY-MM-DD_HHMMSS_摘要.md
```

### 子命令：`list`

1. 使用 `Glob` 扫描 `memory/sessions/*.md`
2. 按时间倒序列出所有文件
3. 输出格式：
   ```
   📂 已保存的会话记录：
   1. 2026-03-20_153022_登录表单验证.md
   2. 2026-03-19_091500_路由重构.md
   3. 2026-03-18_141200_样式调整.md
   ```

### 工具失败兜底

当 `Glob` 因本地 `rg` 缺失、路径异常等原因失败时，改用 `Bash` 执行只读 Python 脚本完成同等扫描：
- 使用 `pathlib.Path.rglob('*.md')` 列出 `memory/sessions/` 下的 session 文件
- 使用 Python 按文件名或修改时间排序，找出最新文件和最早文件
- 只允许读取、排序和统计，不修改、不删除任何 session 文件
- 如需删除超限 session，必须由保存流程按既有数量限制执行，不得由 fallback 脚本直接删除

### 子命令：`load [文件名]`

1. **无参数** — 使用 `Glob` 找到 `memory/sessions/` 下最新的文件，读取它
2. **有参数** — 读取指定文件名（支持模糊匹配，如 `登录` 会匹配包含"登录"的文件）
3. 使用 `Read` 读取文件内容
4. 将内容呈现给用户，并说明：
   ```
   📖 已加载会话上下文喵！
   文件: memory/sessions/YYYY-MM-DD_HHMMSS_摘要.md

   （展示文件内容摘要）

   需要基于这个上下文继续工作吗？
   ```

## 保存流程（核心）

每次保存都执行以下步骤：

### 1. 回顾会话内容

回顾当前会话中的全部工作，提炼关键信息：
- 在做什么任务
- 遇到了什么问题（现象、错误信息）
- 尝试了什么方案
- 得出了什么结论
- 还有什么没做完

### 2. 生成文件名

格式：`YYYY-MM-DD_HHMMSS_任务摘要.md`
- 日期时间取当前时刻
- 任务摘要从会话内容中提炼，控制在 8 字以内
- 示例：`2026-03-20_153022_登录表单验证.md`

### 3. 整理为固定格式

按以下模板填充：
```markdown
# 会话上下文：{任务摘要}

## 任务状态
- **当前阶段**：{研究/开发/测试/调试}
- **完成度**：{百分比}
- **阻塞项**：{无/具体问题}

## 关键结论
{最终结论，不超过 5 条}

## 最后交互摘要
{一句话概括最后一次用户交互}

## 待办事项
- [ ] {待办1}
- [ ] {待办2}
```

注意"最后交互摘要"字段必须填写，用一句话概括最后一次用户交互的内容，供下次定时保存时判断是否有新变化。

### 4. 写入文件

- 目录：`memory/sessions/`
- 目录不存在时使用 `Bash` 执行 `mkdir -p` 自动创建
- 使用 `Write` 写入新文件（每次都是新文件，不覆盖旧的）

## 与 util-memory 的关系

| | util-session | util-memory |
|:---|:---|:---|
| **管理内容** | 会话快照（短期、会话级） | 持久化知识（长期、跨会话） |
| **存储位置** | `memory/sessions/*.md` | `memory/*.md`（MEMORY.md、debugging.md 等） |
| **写入时机** | 手动 save 或定时触发 | 工作中自动积累（按 CLAUDE.md 规则） |
| **清理方式** | 超过 30 个文件时自动删除最早的 | 使用 `/util-memory clean` 手动清理 |

简单判断：**"这次做了什么"存 session，"学到了什么"存 memory。**

## 注意事项

- **精简为王**：每个 session 文件控制在 80 行以内，只记录能帮助接续工作的关键信息
- **结论优先**：重点记录"最终结论"而非"探索过程"
- **错误信息要具体**：保留关键错误信息原文，不要只写"报错了"
- **独立文件**：每次保存生成新文件，历史记录自然保留
- **文件数量限制**：`memory/sessions/` 最多保留 30 个文件，超限时删除最早的
- **手动读取**：不自动读取历史 session，需用户通过 `/util-session load` 主动加载
